# Arquitectura de Base de Datos - Plataforma de Inglés (MVP)

Este documento describe el esquema de la base de datos en Supabase (PostgreSQL) para la primera versión de la plataforma. La estructura está optimizada para manejar progreso granular por ejercicio y un sistema dinámico de recompensas por hitos.

---

## 1. Usuarios y Perfiles

Almacena la información pública y de gamificación vinculada a la autenticación de Supabase (`auth.users`).

| Columna | Tipo | Descripción |
| :--- | :--- | :--- |
| `id` | `UUID` | Llave primaria. Conectada a `auth.users.id`. |
| `first_name` | `Text` | Nombre del usuario (ej. "Naye"). |
| `streak_days` | `Integer` | Días consecutivos de estudio. |
| `created_at` | `Timestamp` | Fecha de creación de la cuenta. |

---

## 2. Estructura de Contenido (Temario)

Organiza los temas de nivel A1, sus lecciones correspondientes y el contenido interactivo.

### `modules`
Bloques temáticos principales.

| Columna | Tipo | Descripción |
| :--- | :--- | :--- |
| `id` | `UUID` | Llave primaria. |
| `title` | `Text` | Título (ej. "Primeros Pasos"). |
| `description` | `Text` | Breve descripción del enfoque. |
| `order` | `Integer` | Orden de aparición en la interfaz (1, 2, 3...). |

### `lessons`
Clases específicas dentro de cada módulo.

| Columna | Tipo | Descripción |
| :--- | :--- | :--- |
| `id` | `UUID` | Llave primaria. |
| `module_id` | `UUID` | Llave foránea `-> modules.id`. |
| `title` | `Text` | Tema de la clase (ej. "Pronombres y Saludos"). |
| `order` | `Integer` | Orden secuencial dentro del módulo. |

### `exercises`
El contenido atómico e interactivo de cada clase.

| Columna | Tipo | Descripción |
| :--- | :--- | :--- |
| `id` | `UUID` | Llave primaria. |
| `lesson_id` | `UUID` | Llave foránea `-> lessons.id`. |
| `type` | `Text` | Categoría: `'voice'`, `'dictation'`, `'writing'`. |
| `content` | `JSONB` | Objeto dinámico. Ej: `{"phrase_en": "Hello", "phrase_es": "Hola"}`. |

---

## 3. Progreso y Recompensas

Registra el avance a nivel micro (ejercicio por ejercicio) y el catálogo de hitos para desbloquear premios.

### `user_lesson_progress`
Guarda el estado exacto de una lección, permitiendo retomarla si se interrumpe.

| Columna | Tipo | Descripción |
| :--- | :--- | :--- |
| `id` | `UUID` | Llave primaria. |
| `user_id` | `UUID` | Llave foránea `-> profiles.id`. |
| `lesson_id` | `UUID` | Llave foránea `-> lessons.id`. |
| `status` | `Text` | Estado actual: `'in_progress'` o `'completed'`. |
| `completed_exercises`| `JSONB` | Arreglo de IDs de ejercicios ya superados (ej. `["id-1", "id-2"]`). |
| `last_accessed_at` | `Timestamp` | Fecha del último acceso para la función "Continuar". |

### `rewards`
Catálogo de hitos dinámicos configurables.

| Columna | Tipo | Descripción |
| :--- | :--- | :--- |
| `id` | `UUID` | Llave primaria. |
| `title` | `Text` | Nombre del premio (ej. "Vale por un picnic"). |
| `condition_type` | `Text` | Regla a evaluar (ej. `'module_completed'`, `'streak_reached'`). |
| `condition_value` | `Text` | Valor a alcanzar (ej. ID del módulo, o `"10"` para 10 días). |
| `icon` | `Text` | Emoji representativo o URL de imagen. |

### `user_rewards`
Registro de los premios que el usuario ha desbloqueado tras cumplir las condiciones.

| Columna | Tipo | Descripción |
| :--- | :--- | :--- |
| `id` | `UUID` | Llave primaria. |
| `user_id` | `UUID` | Llave foránea `-> profiles.id`. |
| `reward_id` | `UUID` | Llave foránea `-> rewards.id`. |
| `unlocked_at` | `Timestamp` | Fecha en la que se cumplió el hito. |
| `is_redeemed` | `Boolean` | `false` por defecto. Cambia a `true` al canjear el premio. |
